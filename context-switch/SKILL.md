# 项目无关请求识别与上下文切换

## 依赖要求

- 无外部依赖
- 无平台限制
- 无 API Key 要求

---

## 执行步骤

### 步骤 1：扫描输入开头

提取用户输入前 10 个字符，匹配以下触发词：

| 中文触发词 | 英文触发词 |
|-----------|-----------|
| `对话`、`聊天`、`闲聊`、`独立要求` | `chat`、`talk`、`independent request` |

另有显式分隔符 `---` 或 `===`，其后内容视为独立请求。

**不匹配任何触发词 → 不切换，继续项目模式。步骤结束。**

### 步骤 2：检查项目特征词

触发词匹配后，检查输入是否包含：

- 项目文件路径（如 `src/main.cpp`）
- 函数名（如 `ThermalDataReader`）
- 技术栈术语（与当前项目相关的库/框架名）

**含项目特征词 → 触发词 + 路径混合场景，走步骤 4。**

### 步骤 3：执行上下文切换

不含项目特征词 → 确认切换：

1. 清空项目相关缓存（文件状态、代码引用、最近操作）
2. 切换至通用模式
3. 直接回答问题，不声明模式变更
4. 不引用任何项目路径、代码、配置

### 步骤 4：混合场景处理

触发词 + 项目路径同时存在：

- 拒绝操作文件
- 警告：`检测到项目文件路径 [路径]，无法在独立模式下操作`
- 如用户坚持 → 请用户去掉触发词后重新发送

### 步骤 5：含糊请求处理

当请求无法判断归属（如 `这个怎么改` 且上下文不明确）：

- 追问：`「[用户原话]」是否指当前项目？`
- 不含预设结论
- 不猜测意图

---

## 失败处理

| 失败场景 | 表现 | 处理 |
|---------|------|------|
| 误触发切换 | 用户说的是项目相关内容但含触发词 | 步骤 2 检查项目特征词应拦截。若已切换，用户纠正时立即切回 |
| 漏判（未切换） | 用户闲聊但没匹配到触发词 | 低危害。在项目模式回答通用问题，回答中不引用项目内容即可 |
| 频繁来回跳转 | 同一对话中项目↔闲聊多次切换 | 短间隔内倾向保持当前模式，避免反复横跳。间隔 < 3 条消息不切换 |
| 危险操作 | `删除所有文件` 含触发词 | 即使触发切换也追加风险提示，不执行 |
| 无法判断 | 输入信息不足以判断归属 | 追问确认，不做默认假设 |

---

## 输出格式

**切换后回答**：
- 直接回答用户问题
- 不引用任何项目内容
- 联网信息标注来源 URL
- 不输出「已退出项目上下文」等模式声明

**追问时**：
```
「[用户原话]」是否指当前项目？
```

---

## 禁止行为

- × 声明「已退出项目上下文」或任何模式变更提示
- × 在通用回答中混入项目路径、代码、配置
- × 对含文件路径的独立请求执行文件操作
- × 来回跳转（3 条消息内不切换）
