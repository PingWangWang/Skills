# 生成标准化提交日志

## 依赖要求

- **外部命令**：`git` 或 `svn` CLI（必须可用）
- **平台限制**：无（Git Bash / Linux / macOS 均可）
- **前置条件**：当前目录必须是 Git 或 SVN 仓库，且有未提交的变更
- **不需要**：API Key、网络连接

---

## 执行步骤

### 步骤 1：检测版本控制系统

执行：
```bash
git rev-parse --git-dir 2>/dev/null
```

- 成功 → Git 仓库，走步骤 2
- 失败 → 检测 `svn info`，成功则走步骤 3
- 都不是 → 输出错误，停止

### 步骤 2：获取 Git 变更

按顺序执行：
```bash
git status                    # 获取变更文件列表
git diff --staged             # 分析已暂存变更（优先）
git diff                      # 补充未暂存变更
git log --oneline -5          # 参考最近 5 条提交风格
```

### 步骤 3：获取 SVN 变更

按顺序执行：
```bash
svn status                    # 获取变更文件列表
svn diff                      # 分析变更内容
svn log --limit 5             # 参考最近 5 条提交风格
```

### 步骤 4：确定提交类型

根据变更内容从下表选类型（只选一个）：

| 类型 | 何时选 |
|------|--------|
| `feat` | 新功能、新接口 |
| `fix` | Bug 修复、边界条件修正 |
| `docs` | 仅文档（README、注释） |
| `style` | 仅格式（缩进、空格、分号） |
| `refactor` | 重构（不改变功能） |
| `perf` | 性能优化 |
| `test` | 仅测试用例 |
| `chore` | 构建、依赖、CI 配置 |

若无法确定 → 默认 `chore`。

若变更涉及多种类型 → 拆分为多个 commit，分别生成。

若有破坏性变更 → 类型后加 `!`（如 `feat!:`）。

### 步骤 5：写 Subject（简短描述）

格式：`类型(作用域): 中文简短描述`

约束：
- 中文撰写（类型和作用域保持英文小写）
- ≤50 字符
- 无句号
- 简洁，不写"了"等多余字词

作用域可选，指受影响的模块/包名（英文小写）。

### 步骤 6：写 Body（详细描述，可选）

满足以下任一条件 → 必须写 Body：
- 变更原因不显而易见
- 涉及多个文件且需要说明关联
- 有破坏性变更

格式：
- 空一行与 Subject 分隔
- 中文撰写
- 说 Why（为什么改）和 How（怎么改）
- 每行 ≤80 字符，手动换行

极简变更（typo 修复、一行改动）→ 省略 Body。

### 步骤 7：写 Footer（可选）

关联工单或声明破坏性变更时写。空一行与 Body 分隔。

### 步骤 8：输出日志文本

只输出日志文本本身。禁止输出任何其他内容。

## 失败处理

| 失败场景 | 表现 | 处理 |
|---------|------|------|
| 不在版本控制目录 | `git rev-parse` 和 `svn info` 都失败 | 输出：`错误：当前目录不是 Git 或 SVN 仓库，无法生成提交日志` |
| 无变更 | `git status` 或 `svn status` 显示干净 | 输出：`无待提交的变更` |
| 无法确定类型 | 变更涉及多种类型 | 分别生成多个 commit message，标注建议拆分为 N 个提交 |
| git/svn 命令不可用 | command not found | 输出：`错误：未找到 git/svn 命令，请确认已安装` |
| diff 内容无法解析 | 二进制文件或编码异常 | 跳过该文件，基于文件名和路径推断变更意图 |
| Subject 超过 50 字符 | 描述太长 | 将多余内容移入 Body，Subject 保留核心动词+对象 |

---

## 输出格式

**成功时**：只输出日志文本，不带任何包装：
```
feat(auth): 新增 OAuth2 登录流程

原登录仅支持用户名密码。接入 OAuth2 以支持
企业 SSO，扩展 token 验证中间件处理新流程。

Closes: JIRA-4321
```

**失败时**：输出一行错误原因（见上表）。

**禁止输出**：
- × `git commit -m "..."` 命令
- × 「以下是提交日志」等开场白
- × 「建议执行...」等后续建议
- × 任何形式的提交操作执行

---

## 禁止行为

- × 执行 `git commit` 或任何提交操作
- × 写入 `.git/COMMIT_EDITMSG`
- × 输出英文 Subject 或 Body（类型和作用域除外）
- × 输出开场白 / 结束语 / 命令建议
- × 在非仓库目录下猜测变更
